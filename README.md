# jeanpierreboutros.com

Personal portfolio site for **Jean-Pierre Boutros** — live & studio sound engineer (Beirut / Jounieh, Lebanon), founder of StudioJPB and Zynforge Audio.

🌐 **Live:** https://www.jeanpierreboutros.com

A single-page static site: dark/gold editorial design, no framework, no build step required to serve. Fonts load from Google Fonts; everything else is self-hosted.

---

## Stack

- **HTML / CSS / vanilla JS** — zero dependencies, no bundler at runtime
- **Fonts:** Fraunces (serif) + Hanken Grotesk (sans) via Google Fonts
- **Generator:** `build.py` renders `index.html` from structured data (artists, gallery, TV, bio)
- Hosted on **HostGator** (cPanel), deployed from this repo via **Git Version Control**

## Structure

```
.
├── index.html          # generated by build.py — do not hand-edit content
├── styles.css          # design system + components (hand-authored)
├── main.js             # nav, scroll FX, reveal, name-wall popovers, lightbox
├── build.py            # regenerates index.html from data
├── robots.txt
├── sitemap.xml
├── favicon.png         # JPB monogram on near-black
├── jpb-monogram.png    # nav + footer mark

```

## Editing content

Content lives in **`build.py`** as Python lists, then is rendered into `index.html`:

| What | Where in `build.py` |
|------|---------------------|
| Artists / clients (name, role, bio) | `clients = [...]` |
| Gallery images + sizing (`tall`/`wide`/`normal`) | `gallery_all = [...]` |
| TV / broadcast cards | `tv = [...]` |
| Zynforge apps | `zyn = [...]` |
| Skills list | `skills = [...]` |
| Hero waveform bars | `wave = [...]` |
| SEO / Open Graph / JSON-LD | `jsonld = {...}` and the `INDEX` head |

After editing, regenerate:

```bash
python3 build.py        # rewrites index.html in place
```

> Gallery entries are only rendered if the matching file exists in `photos/`, so a missing image is skipped rather than shown broken.

- **Styling** → edit `styles.css` directly (token-based; see below).
- **Behavior** (menu, lightbox, scroll) → edit `main.js`.

## Images

- Stored in `photos/`, compressed to ≤ ~1920 px, quality ~70–85.
- Orientation is **baked into the pixels** (EXIF stripped) so they render identically in every browser.
- New images: drop into `photos/` using the names referenced in `build.py` (or add a new entry), then `python3 build.py`.

```bash
# compress a new phone photo
sips -Z 1920 -s format jpeg -s formatOptions 70 IN.jpg --out photos/NAME.jpg
```

## Design system

Defined as CSS custom properties at the top of `styles.css`:

- **Color** — `--bg`, `--surface*`, `--text*`, `--muted*`, `--accent` (gold `#c8a96e`), plus `--*-rgb` channel vars for `rgb(var(--x) / alpha)`
- **Type** — fluid display tokens (`--fs-hero`, `--fs-h2`) + step ramp (`--fs-3xs … --fs-4xl`)
- **Spacing** — `--space-1 … --space-8`, `--pad`, `--section-pad`
- **Motion** — `--t-fast / --t-card / --t-med / --t-slow`
- **Radius** — `--radius-sm / md / full`

Accessibility: keyboard `:focus-visible` rings, WCAG-AA text contrast, `prefers-reduced-motion` honored, real `<button>`s with ARIA, lightbox keyboard controls (Esc / ← / →).

## Deployment (HostGator via Git Version Control)

The site auto-copies into the web root on deploy via **`.cpanel.yml`** (document root is `public_html`).

**One-time:** cPanel → *Git Version Control* → *Create* → clone
`https://github.com/jeanpierreboutros-lang/jeanpierreboutros.com.git`
(repository path: leave default — **not** the docroot).

**Every update:**

1. Push to `main` on GitHub.
2. cPanel → *Git Version Control* → repo → *Pull or Deploy* → **Update from Remote** → **Deploy HEAD Commit**.
3. Hard-refresh https://www.jeanpierreboutros.com.

`.cpanel.yml` copies the runtime files (HTML/CSS/JS, images, icons, robots, sitemap) into `public_html`; `build.py`, `README.md`, and `.git` are not published.
